# Contributing to University Repo

We love your input! We want to make contributing to this project as easy and transparent as possible.

## How to Contribute

### 1. Issues
- Check the **Projects** tab for available tasks.
- If you find a bug or have a feature idea, open a new **Issue**.

### 2. Workflow
- Always create a new branch for your task: `type/short-description`
- Follow the commit naming convention: `type(scope): description`
- Before pushing, make sure to sync with main: `git fetch origin` then `git rebase origin/main`

### 3. Pull Requests
- Keep PRs small and focused on one task.
- Use `Closes #ISSUE_NUMBER` in your PR description.
- Wait for a reviewer to approve before merging.

### 4. Branch Naming & Commit Types
- `feat`: New features
- `fix`: Bug fixes
- `docs`: Documentation
- `refactor`: Code changes that don't fix bugs or add features
- `chore`: Configuration and tooling
