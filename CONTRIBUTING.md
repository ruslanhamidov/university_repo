# Contributing

Thank you for contributing to this project.

## Fork the Repository

1. Fork the repository on GitHub.
2. Clone your fork locally.
3. Add the original repository as an upstream remote.

## Create a Branch

Before starting any task, create a new branch.

Branch naming convention:

type/short-description

Examples:

- feat/user-login
- fix/navbar-overflow
- docs/update-readme
- chore/update-dependencies

## Commit Rules

Commit messages should follow:

type(scope): short description

Examples:

- feat(auth): add login feature
- fix(ui): resolve mobile layout issue
- docs(readme): update setup instructions

## Pull Request Rules

1. Push your branch to GitHub.
2. Open a Pull Request against main.
3. Use a descriptive PR title.
4. Link the related issue using:

Closes #ISSUE_NUMBER

5. Request a review from at least one teammate.

## Getting a Review

- Wait for reviewer feedback.
- Make requested changes if needed.
- Do not merge your own Pull Request unless instructed by the team.
