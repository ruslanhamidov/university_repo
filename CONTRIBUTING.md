\# Contributing Guide



Thank you for contributing to this project.



\## Before You Start



\* Read the project documentation.

\* Check existing issues before creating a new one.

\* Create a separate branch for your work.



\## Branch Naming Convention



Use the following format:



```text

type/short-description

```



Examples:



```text

feat/add-login-page

docs/update-readme

fix/navbar-bug

```



\## Commit Message Convention



Use clear and descriptive commit messages.



Format:



```text

type(scope): short description

```



Examples:



```text

feat(auth): add login functionality

docs(readme): update installation steps

fix(api): handle null response

```



\## Pull Requests



Before opening a pull request:



\* Make sure your code works correctly.

\* Make sure documentation is updated if needed.

\* Keep pull requests focused on a single task.



\### Pull Request Template



Include:



\* What was changed

\* Why it was changed

\* How it can be tested



\## Code Reviews



All pull requests should be reviewed before merging.



Reviewers should:



\* Check correctness

\* Check readability

\* Check project conventions



\## Reporting Issues



When creating an issue:



\* Use a clear title

\* Explain the problem

\* Provide steps to reproduce if applicable



\## Communication



Be respectful and constructive when discussing changes.



Thank you for helping improve this project.
