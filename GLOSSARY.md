    # Git and GitHub Glossary

    ## 1. Repository (Repo)
    Layihənizin fayllarının və versiya tarixçəsinin saxlandığı yer.
    *Nümunə:* `git clone https://github.com/user/repo.git`

    ## 2. Commit
    Fayllarınızın müəyyən bir andakı vəziyyətinin şəklidir.
    *Nümunə:* `git commit -m "feat(auth): login elave olundu"`

    ## 3. Branch
    Layihənizin əsas koduna təsir etmədən yeni funksiyalar üzərində işləməyə imkan verən paralel versiya.
    *Nümunə:* `git checkout -b feature/yeni-dizayn`

    ## 4. Fork
    Başqasının repozitoriyasının şəxsi GitHub hesabınızda olan surəti.
    *Nümunə:* GitHub-da 'Fork' düyməsinə klikləmək.

    ## 5. Clone
    Uzaqdakı repozitoriyanın kompüterinizə yerli surətini çıxarmaq.
    *Nümunə:* `git clone <url>`

    ## 6. Pull Request (PR)
    Dəyişikliklərinizin yoxlanılması və əsas layihəyə birləşdirilməsi üçün təklif.
    *Nümunə:* GitHub-da 'Compare & pull request' düyməsinə klikləmək.

    ## 7. Merge
    Bir branch-dakı dəyişiklikləri başqa bir branch-a tətbiq etmək prosesi.
    *Nümunə:* `git merge feature/yeni-funksiya`

    ## 8. Remote
    İnternetdə və ya şəbəkədə yerləşən repozitoriya versiyası.
    *Nümunə:* `origin` və ya `upstream`.

    ## 9. Staging Area
    Dəyişiklikləri commit etməzdən əvvəl hazırladığınız yer.
    *Nümunə:* `git add GLOSSARY.md`

    ## 10. Conflict
    İki branch arasındakı fərqləri Git-in avtomatik həll edə bilmədiyi vəziyyət.
    *Nümunə:* Eyni sətirdə iki fərqli dəyişiklik edilməsi.
