\# Git and GitHub Glossary



This glossary explains common Git and GitHub terms used in this project.



\## Repository



A repository is a project folder tracked by Git. It contains files, folders, commits, and project history.



\## Git



Git is a version control system. It helps developers track changes in files and work on the same project without losing previous versions.



\## GitHub



GitHub is an online platform for hosting Git repositories. It allows developers to collaborate, review code, create issues, and manage pull requests.



\## Fork



A fork is a personal copy of another repository. It allows you to make changes without affecting the original project directly.



\## Clone



Cloning means downloading a repository from GitHub to your local computer.



\## Branch



A branch is a separate line of development. Developers use branches to work on features or fixes without changing the main branch directly.



\## Commit



A commit is a saved snapshot of changes in a repository. Each commit should describe what was changed.



\## Commit Message



A commit message is a short explanation of what was changed in a commit.



\## Pull Request



A pull request is a request to merge changes from one branch into another branch. It is used for code review and discussion before merging.



\## Merge



Merge means combining changes from one branch into another branch.



\## Merge Conflict



A merge conflict happens when Git cannot automatically combine changes because two people edited the same part of a file.



\## Issue



An issue is a task, bug report, or discussion item in GitHub. Teams use issues to organize work.



\## Kanban Board



A Kanban board is a visual board used to track tasks. Common columns are To Do, In Progress, Review, and Done.



\## Main Branch



The main branch is the primary branch of a repository. Final and stable changes are usually merged into this branch.



\## Remote



A remote is the online version of a repository, usually stored on GitHub.



\## Pull



Pull means downloading the latest changes from the remote repository to your local computer.



\## Push



Push means uploading your local commits to the remote repository on GitHub.



\## Review



Review is the process of checking someone’s pull request before it is merged.



\## Collaborator



A collaborator is a person who has permission to work on a GitHub repository.
