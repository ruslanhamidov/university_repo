# Git & GitHub Glossary

## Branch
A branch is a separate line of development used to work on features without affecting the main code.
Example: `git checkout -b feat/login`

## Commit
A commit is a saved snapshot of your changes in Git.
Example: `git commit -m "add login page"`

## Repository
A repository is a project folder that tracks all files and their history.
Example: A GitHub repo storing your project code.

## Fork
A fork is your personal copy of someone else's repository.
Example: Forking a project to make changes independently.

## Clone
Cloning means downloading a repository to your local computer.
Example: `git clone https://github.com/user/repo.git`

## Origin
Origin is the default remote pointing to your forked repository.
Example: `git push origin main`

## Upstream
Upstream is the original repository you forked from.
Example: `git pull upstream main`

## Staging Area
The staging area is where changes are prepared before committing.
Example: `git add file.js`

## HEAD
HEAD points to your current commit or branch in Git.
Example: HEAD moves when you switch branches.

## Rebase
Rebase moves your commits on top of another branch to keep history clean.
Example: `git rebase main`

## Merge
Merge combines changes from one branch into another.
Example: merging a feature branch into main.

## Pull Request
A pull request is a request to merge your changes into another branch.
Example: opening PR after pushing code.

## Conflict
A conflict happens when Git cannot automatically merge changes.
Example: two people edit the same line.

## Push
Push uploads your commits to GitHub.
Example: `git push origin feature`

## Pull
Pull downloads changes from GitHub to your local repo.
Example: `git pull origin main`
