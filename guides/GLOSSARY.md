# Git & GitHub Glossary

## Branch
Definition: A branch is a separate line of development that lets you work on changes without affecting the main branch.

Example: I created a feature branch to add a new glossary file.

## Commit
Definition: A commit is a saved snapshot of changes in a repository.

Example: I made a commit after finishing the glossary.

## Repository
Definition: A repository is a project's storage location that contains files and version history.

Example: The team stores all project files in one repository.

## Clone
Definition: Cloning creates a local copy of a remote repository on your computer.

Example: I cloned the repository before making changes.

## Pull Request (PR)
Definition: A pull request is a request to review and merge changes into another branch.

Example: I opened a PR for my glossary update.

## Merge
Definition: A merge combines changes from one branch into another.

Example: The feature branch was merged into main.

## Rebase
Definition: Rebasing moves your commits onto a new base commit to keep history cleaner.

Example: I rebased my branch onto the latest main branch.

## Staging Area
Definition: The staging area is where changes are prepared before being committed.

Example: I added GLOSSARY.md to the staging area with git add.

## HEAD
Definition: HEAD is a pointer to the commit you are currently working on.

Example: HEAD points to the latest commit on my branch.

## Upstream
Definition: Upstream refers to the branch or repository that your local branch tracks.

Example: I pulled updates from the upstream main branch.
